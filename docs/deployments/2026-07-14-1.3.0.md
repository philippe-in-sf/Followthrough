# Deployment Audit - 2026-07-14 - 1.3.0

- Previous version: 1.2.17
- New version: 1.3.0
- Version increment: minor release
- Deployment target: production after merge to `main`
- Audit note: this release records the required deployment version bump for automatic database backups and the user-facing meeting notes browser.

## Release Contents

- Automatic SQLite backups run from the server process with configurable interval, directory, and retention count.
- Backup runs create snapshot files and append JSONL manifest entries for deployment auditing.
- Users can open a Notes section to see meeting notes from meetings they created or attended by week, month, or custom date range.

## Verification Notes

- `git diff --check` passed before release preparation.
- A direct SQLite `VACUUM INTO` smoke check produced a readable backup snapshot.
- Full local Vitest and TypeScript checks could not complete in the managed Codex environment because worker/compiler processes stalled before useful diagnostics.

## Deployment Notes

- The deploy runner requires a clean local `main` branch matching `origin/main`.
- This release should be deployed only after the PR merges to `main`, then rerun `npm run deploy -- production` from a clean synced `main`.
